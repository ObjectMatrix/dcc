### 1.1. Course Overview
Students build a wrist-mounted gesture controller that wirelessly drives a 3-to-4 joint robotic arm. By the end of this course, every student will have soldered a working wearable, written firmware for two ESP32 microcontrollers communicating over ESP-NOW, and tuned a proportional controller so that tilting their wrist makes a physical arm mirror their motion in real time.

### 1.1 Big Idea
The Core Insight
A robot is just a loop: SENSE → DECIDE → ACT.
The MPU-6050 on your wrist is the SENSE. The ESP-NOW radio is the pathway.
The PID controller is the DECIDE. The servo motors are the ACT.
Once you build this loop, every robot you ever make will follow the same architecture.

### 1.2 Learning Objectives
By the end of all three sessions, students will be able to:
	•	Read I2C sensor data — poll an MPU-6050 and extract pitch/roll angles using complementary filtering
	•	Control servos from code — map sensor angles to pulse-width values and drive SG90 motors
	•	Transmit data wirelessly — use ESP-NOW peer-to-peer protocol to send structs from one ESP32 to another
	•	Apply PID-lite control — implement a proportional (P) controller and understand why adding I and D terms smooths motion
	•	Debug embedded systems — use Serial Monitor, LED indicators, and systematic fault isolation to find hardware/firmware bugs

### 1.3 Concept Map
Concept
Where It Appears in This Project

### I2C Protocol
ESP32 reads MPU-6050 sensor data

### Euler Angles (Pitch/Roll)
Sensor fusion converts raw accel+gyro into tilt angles

### Complementary Filter
Blends accel and gyro to reduce noise and drift

### ESP-NOW Protocol
Sender ESP32 streams angles to Receiver ESP32 at ~50 Hz

### PWM & Servo Control
Receiver maps angle values to microsecond servo pulses

### Proportional Controller
Servo angle = P × error; reduces overshoot vs direct mapping

### Struct Serialization
Angle data packed into a C struct for wireless transmission

### MAC Address Pairing
Two ESP32 boards discover each other before streaming
