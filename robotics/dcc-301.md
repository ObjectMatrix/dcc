# DCC-301: Weather Buddy (DHT11 Sensor)

![dcc-301](./images/300.png)

This core lesson introduces sensors. Students read real-world temperature and humidity using a DHT11 and display values live.

## Core Flow (same in every DCC core lesson)

1. Build the circuit.
2. Run the starter code.
3. Observe what changes in real life.
4. Explain the concept in plain words.
5. Try a challenge.

## Big Idea

Sensors convert physical conditions into numbers. Here, the DHT11 turns air temperature and humidity into digital data that ESP32 can read.

## What You Will Build

A room weather monitor that prints:

- Temperature in Celsius and Fahrenheit
- Humidity percentage

## What You Will Learn

- How to wire and read a digital sensor
- How loops collect repeated measurements
- How to convert Celsius to Fahrenheit
- How to use basic error handling
- How environmental data changes in real time

## Wiring

| DHT11 pin | ESP32 pin |
| --- | --- |
| VCC | 3V3 |
| DATA | GPIO 4 |
| GND | GND |

Important: use 3V3, not 5V.

## Starter Code

```python
from machine import Pin   # lets our code use the ESP32's pins
import dht                # a helper that knows how to talk to DHT sensors
import time               # lets us pause the program

sensor = dht.DHT11(Pin(4))   # our DHT11 sensor is wired to pin 4

while True:               # do this forever
    try:                  # "try" means: attempt this, but don't crash if it fails
        sensor.measure()          # tell the sensor: take a fresh reading now
        temp_c = sensor.temperature()   # get the temperature in Celsius
        hum = sensor.humidity()         # get the humidity as a percent
        temp_f = temp_c * 9 / 5 + 32    # math to change Celsius into Fahrenheit

        # print a neat line. {:.1f} means show 1 number after the dot (like 23.4)
        print("Temp: {:.1f} C / {:.1f} F   Humidity: {:.1f} %".format(
            temp_c, temp_f, hum
        ))
    except OSError:       # if the sensor didn't answer (loose wire, too fast)...
        print("Sensor read failed, retrying...")   # ...just say so and keep going

    time.sleep(2)         # wait 2 seconds. The DHT11 is slow and needs a rest
```

## Optional Cloud Extension (Teacher Demo)

Do not hardcode real secrets in class materials. Use placeholders.

```python
import network        # lets the ESP32 join WiFi
import urequests      # lets the ESP32 send data to a website

WIFI_SSID = "YOUR_WIFI_NAME"          # <-- your WiFi name
WIFI_PASSWORD = "YOUR_WIFI_PASSWORD"    # <-- your WiFi password
THINGSPEAK_KEY = "YOUR_THINGSPEAK_WRITE_KEY"   # <-- your secret upload key (keep private)

# Connect once
wlan = network.WLAN(network.STA_IF)   # turn the ESP32 into a WiFi device
wlan.active(True)                     # switch the WiFi radio on
wlan.connect(WIFI_SSID, WIFI_PASSWORD)   # join your network
while not wlan.isconnected():         # keep waiting until we're connected
    time.sleep(1)

# Inside loop after temp/hum are read
# Build a web address that carries our two numbers, then visit it to save them online.
url = "https://api.thingspeak.com/update?api_key={}&field1={}&field2={}".format(
    THINGSPEAK_KEY, temp_f, hum
)
r = urequests.get(url)   # send the data up to the ThingSpeak website
r.close()                # close the connection to free up memory
```

- Basic formulas and print formatting make data easier to read.

## Troubleshooting

| Problem | Likely Cause | Fix |
| --- | --- | --- |
| ETIMEDOUT or OSError | Loose DATA wire or read too fast | Recheck wires and keep 2 second delay |
| Unrealistic values | Wrong sensor model in code | Ensure `dht.DHT11(Pin(4))` |
| No output | Program not running | Click Run and check shell |
| Always failing reads | Wrong power pin | Move VCC to 3V3 |
