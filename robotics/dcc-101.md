# DCC-101: Intro to Circuits and GPIO (ESP32)

## Big Idea

An ESP32 pin is like a tiny switch controlled by code. When we set a pin HIGH, voltage appears on the pin. When we set it LOW, voltage disappears. That simple idea powers almost every robot project.

## What You Will Build

You will build one LED circuit and complete three coding mini-labs:

1. Blink (ON/OFF)
2. Breathe (PWM fading)
3. SOS signal (timing pattern)

## What You Will Learn

- What GPIO means and how output pins work
- Why we use a resistor with an LED
- How loops repeat actions forever
- How PWM can make digital hardware feel analog
- How timing creates recognizable patterns

## Wiring

Use GPIO 18 in this lesson so wiring and code match exactly.

| ESP32 | Connect to |
| --- | --- |
| GPIO 18 | 220 ohm resistor -> LED long leg (+) |
| GND | LED short leg (-) |

Safety rule: never connect an LED directly to a GPIO pin without a resistor.

## Starter Code

### Part A: Blink

```python
from machine import Pin   # Pin lets our code control the ESP32's metal pins
import time               # time lets us pause the program for a moment

LED_PIN = 18              # our LED is wired to pin number 18
led = Pin(LED_PIN, Pin.OUT)   # tell the board: pin 18 is an OUTPUT (it sends power out)

while True:               # "while True" means do this forever, over and over
    led.value(1)          # 1 = turn the LED ON (send power to the pin)
    time.sleep(1)         # wait for 1 second while it stays on
    led.value(0)          # 0 = turn the LED OFF (stop the power)
    time.sleep(1)         # wait for 1 second while it stays off
```

### Part B: Breathe (PWM)

```python
from machine import Pin, PWM   # PWM lets us make the LED dim, not just ON/OFF
import time

LED_PIN = 18
led = PWM(Pin(LED_PIN), freq=5000)   # blink pin 18 super fast (5000 times a second)
                                     # so fast your eyes see brightness, not blinking

def set_brightness(level_0_to_255):  # a helper that sets how bright the LED is
    # 0 = off, 255 = fully bright. The board wants a bigger number (0 to 1023),
    # so we scale our easy 0-255 number up to what the board expects.
    duty_10bit = int(level_0_to_255 * 1023 / 255)
    led.duty(duty_10bit)             # actually change the brightness

while True:
    for b in range(0, 256):          # count up from 0 to 255 = slowly get brighter
        set_brightness(b)
        time.sleep_ms(8)             # tiny pause so the fade is smooth (8 milliseconds)
    for b in range(255, -1, -1):     # count back down 255 to 0 = slowly get dimmer
        set_brightness(b)
        time.sleep_ms(8)
```

### Part C: SOS Pattern

```python
from machine import Pin
import time

LED_PIN = 18
UNIT = 200                # one "time unit" is 200 milliseconds. Change this to go faster/slower
led = Pin(LED_PIN, Pin.OUT)

def dot():                # a dot = a SHORT flash
    led.value(1)
    time.sleep_ms(UNIT)       # on for 1 unit
    led.value(0)
    time.sleep_ms(UNIT)       # off for 1 unit (small gap after)

def dash():               # a dash = a LONG flash
    led.value(1)
    time.sleep_ms(UNIT * 3)   # on for 3 units (3 times longer than a dot)
    led.value(0)
    time.sleep_ms(UNIT)       # off for 1 unit

def letter_gap():         # extra pause between letters
    time.sleep_ms(UNIT * 2)

def word_gap():           # bigger pause between whole words
    time.sleep_ms(UNIT * 6)

while True:
    dot(); dot(); dot(); letter_gap()      # S = three dots
    dash(); dash(); dash(); letter_gap()   # O = three dashes
    dot(); dot(); dot(); word_gap()        # S = three dots, then a long rest
```

## Explain the Concept

- Blink: one pin acts like a switch.
- Breathe: PWM changes ON-time percentage, so the LED appears brighter or dimmer.
- SOS: timing patterns are data.

## Troubleshooting

| Problem | Likely Cause | Fix |
| --- | --- | --- |
| LED never turns on | LED polarity reversed | Swap LED legs |
| LED always on | Wire in wrong row | Recheck breadboard rows |
| Board resets | Short circuit | Unplug, inspect wiring, reconnect |
| PWM looks like blink only | Wrong pin or code not running | Verify GPIO 18 and rerun script |
