# DCC-201: WiFi-Controlled LED (Intro to IoT)

![dcc-201](./images/200.jpeg)

This core lesson introduces Internet of Things (IoT). Students use a phone browser to control a real LED connected to ESP32.

## Core Flow (same in every DCC core lesson)

1. Build the circuit.
2. Run the starter code.
3. Observe what changes in real life.
4. Explain the concept in plain words.
5. Try a challenge.

## Big Idea

In IoT, one device asks and another device answers. Here, the phone is the client and ESP32 is the server. A URL path like `/on` becomes a hardware action.

## What You Will Build

You will host a tiny webpage on the ESP32 with two buttons:

- LED ON
- LED OFF

## What You Will Learn

- How ESP32 connects to local WiFi
- What an IP address is
- How HTTP GET requests work
- How URL routing controls GPIO
- How software and hardware connect

| GND | LED short leg (-) |

## Starter Code

```python
import network      # lets the ESP32 join a WiFi network
import socket       # lets the ESP32 listen for web page visits
import machine      # lets us control the pins
import time         # lets us pause the program

WIFI_SSID = "YOUR_WIFI_NAME"        # <-- put your WiFi name here
WIFI_PASSWORD = "YOUR_WIFI_PASSWORD"  # <-- put your WiFi password here
PORT = 8080         # the "door number" our web page lives behind
LED_PIN = 18        # the LED is wired to pin 18

led = machine.Pin(LED_PIN, machine.Pin.OUT)   # pin 18 is an output (sends power out)

def connect_wifi():                 # this function joins the WiFi
    wlan = network.WLAN(network.STA_IF)   # turn the ESP32 into a WiFi device
    wlan.active(True)               # switch the WiFi radio on
    wlan.connect(WIFI_SSID, WIFI_PASSWORD)   # try to join your network
    print("Connecting", end="")
    for _ in range(20):             # keep checking up to 20 times
        if wlan.isconnected():      # did we get on the network yet?
            break                   # yes! stop waiting
        print(".", end="")          # print a dot to show we're still trying
        time.sleep(0.5)             # wait half a second, then check again
    if wlan.isconnected():
        ip = wlan.ifconfig()[0]     # our address on the network (like a house number)
        print("\nConnected! Open http://{}:{}".format(ip, PORT))
        return ip                   # hand the address back so we can use it
    print("\nWiFi failed")
    return None

def html_page(status):              # builds the web page the phone will see
    # This is just text that describes a simple page with two buttons.
    return """HTTP/1.1 200 OK\r\nContent-Type: text/html\r\n\r\n
<!doctype html>
<html><body style='font-family:sans-serif;text-align:center;padding:30px;'>
<h1>DCC-201 LED Control</h1>
<p>LED is <b>{}</b></p>
<a href='/on'><button style='padding:12px 20px;'>ON</button></a>
<a href='/off'><button style='padding:12px 20px;'>OFF</button></a>
</body></html>
""".format(status)

def start_server(ip):               # waits for the phone to visit and reacts
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)   # make a "mailbox" for visits
    s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)  # let us reuse the door number
    s.bind((ip, PORT))              # put the mailbox at our address and door number
    s.listen(3)                     # start listening for visitors
    status = "OFF"                  # remember whether the LED is on or off
    led.value(0)                    # start with the LED off
    print("Server ready")

    while True:                     # keep serving forever
        conn, _ = s.accept()        # wait here until a phone connects
        req = conn.recv(1024).decode("utf-8")   # read what the phone asked for
        path = "/"
        if "GET " in req:           # pull out the part after GET, like "/on"
            path = req.split("GET ")[1].split(" ")[0]

        if path == "/on":           # if the phone visited /on
            led.value(1)            # turn the LED on
            status = "ON"
        elif path == "/off":        # if the phone visited /off
            led.value(0)            # turn the LED off
            status = "OFF"

        conn.send(html_page(status).encode())   # send the page back to the phone
        conn.close()                # hang up until the next visit

ip = connect_wifi()                 # step 1: get on WiFi
if ip:                              # step 2: if that worked...
    start_server(ip)                # ...start listening for the phone
```

## Explain the Concept

- ESP32 joins WiFi and gets a local IP address.
- Browser sends a request like `/on`.
- Code reads that path and flips a GPIO pin.

## Troubleshooting

| Problem | Likely Cause | Fix |
| --- | --- | --- |
| Phone cannot open page | Wrong IP or not on same WiFi | Confirm IP in shell and same network |
| WiFi never connects | Wrong SSID/password | Recheck credentials |
| Buttons show page but LED does not change | Wiring mismatch | Recheck GPIO 18 and GND |
| Browser says site cannot be reached | Script stopped with error | Rerun and read shell output |
